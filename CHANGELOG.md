# Changelog

## v2.4.6

- 将四组创意池统一扩充到 100 条，并整体改成更偏青春少女、清爽轻盈、真实日常的词条。
- 调整日程线索引导，明确避免显老、轻熟、姨感、成熟职场照和过重妆造。
- 细化分时段默认穿搭与自拍提示，让晨起、出门、白天、回家和夜间都更偏年轻自然的少女感。
- 同步更新默认 `prompt_template` 文案，改成当前真实生效的“6 行线索”模式说明。

## v2.4.5

- 屏蔽旧版 AstrBot live schema 结构不兼容导致的 provider 刷新 warning，不再污染运行日志。
- 增加 `schedule generation mode=no_json_guidance` 明确日志，便于确认云端实际跑到的是“无 JSON 依赖”主链。

## v2.4.4

- 重写 `generator.py` 主生成链，不再要求模型直接输出完整 JSON 结构。
- 改为“模型只提供少量文本线索，完整日程与六段时段由代码本地组装”。
- 当模型返回空文本、占位废话或普通自然语言时，仍会生成 `status=ok` 的固定窗口日程。
- 保留自选模型能力，并让线索继续影响全天摘要、白天活动、晚间状态和自拍氛围。

## v2.4.3

- 修复不同 AstrBot 版本中 `metadata.config` 结构不一致时，实时刷新 `schedule_provider_id` 下拉报 warning 的问题。
- 兼容 `dict`、`list` 与带 `schema` 属性的配置对象。

## v2.4.2

- 修复分时段分支里 `generator.py` 使用 `@compat_dataclass` 时，辅助函数定义顺序错误导致导入阶段 `NameError`。

## v2.4.1

- 继续细化分时段穿搭字段，扩展到上装、下装、外搭、鞋履、配饰、发型、妆面、自拍姿态和自拍光线。
- 补充旧窗口数据自动补齐逻辑，旧缓存也能升级到新字段。

## v2.4.0

- 升级为“刷新锚点起算 24 小时固定窗口”模型，同一窗口内不再反复变化。
- 新增六段分时段结构，覆盖起床、通勤、白天工作、下班返程、到家放松和夜间休息。
- 每段包含穿搭、活动、地点、情绪、自拍场景、改图提示和文案线索。

## v2.3.7

- 修复部分 AstrBot 版本里 `metadata.config.schema` 为 `list` 时，动态刷新 provider 下拉会报 `'list' object has no attribute 'get'`。
- 增加日程生成时的实际 provider 日志，便于确认是否真的走到选中的模型。

## v2.3.6

- 主线版本支持自定义选择日程生成所使用的聊天模型。
- 新增 `schedule_provider_id` 配置项，留空时继续跟随当前会话模型。

## v2.3.2

- 增强日程生成阶段的 JSON 容错解析，兼容 fenced code block、伪 JSON、花括号片段和 key-value 文本输出。

## v2.3.1

- 新增根目录兼容模块，降低 GitHub 安装流程对子目录处理不完整时的导入失败概率。

## v2.3.0

- 调整插件公开信息，统一为增强版仓库信息。
- 更新作者与仓库链接，并清理旧公开群号与二维码信息。
